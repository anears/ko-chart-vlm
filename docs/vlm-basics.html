VLM 기초 — 이 프로젝트에 필요한 만큼만

ko-chart-vlm 프로젝트를 진행하는 데 필요한 최소한의 Vision-Language Model 개념 정리.

VLM이 뭔가

Vision-Language Model = 이미지(+텍스트)를 입력받아 텍스트를 출력하는 LLM. 본질은 "LLM이 이미지 토큰도 읽을 수 있게 확장한 것"이다. 이미지를 별도의 분류기로 처리하는 게 아니라, 이미지를 토큰 시퀀스로 바꿔 텍스트 토큰과 한 줄로 이어붙여 LLM에 넣는다.

구조 — 최신 오픈 VLM은 거의 전부 이 3단 구성

이미지픽셀
→
① Vision EncoderViT — 패치를 벡터로
→
② ProjectorMLP — LLM 공간으로 변환
→
③ LLM이미지 토큰 + 질문 토큰 → 답변 생성
  1. Vision Encoder: 이미지를 패치(예: 14×14px 조각)로 잘라 패치마다 벡터 하나씩 만든다. 보통 CLIP/SigLIP 계열 ViT.
  2. Projector: 비전 벡터를 LLM의 임베딩 공간으로 변환하는 작은 MLP. (LLaVA의 핵심 발견 — 이 단순한 연결로 충분하다)
  3. LLM: 이미지 토큰을 마치 외국어 단어처럼 취급하며 평소처럼 다음 토큰을 생성한다.

계보 — 한 줄씩만

모델기여
CLIP (2021)이미지·텍스트를 같은 벡터 공간에 매핑(대조학습). 생성은 못 하지만 이후 모든 VLM의 "눈"이 됨
BLIP-2 (2023)얼려둔 ViT와 얼려둔 LLM을 Q-Former라는 어댑터로 연결 — "붙이면 된다"를 증명
LLaVA (2023)어댑터를 MLP 하나로 단순화 + GPT-4로 만든 instruction 데이터로 SFT. 오픈 VLM 표준 레시피 확립
Qwen-VL 계열 (2023~)dynamic resolution(이미지 해상도에 비례해 토큰 수 가변), M-RoPE. 현재 오픈 VLM 사실상의 표준 베이스

학습은 3단계

  1. Alignment pretraining — 이미지-캡션 쌍 수억 장으로 projector(±비전 인코더)를 LLM에 정렬
  2. Instruction tuning (SFT) — "이미지-질문-답" 대화 데이터로 지시를 따르게 학습
  3. Preference tuning (선택) — DPO/RLHF로 선호 최적화
이 프로젝트가 하는 일: 이미 1·2단계를 거친 공개 모델(Qwen3-VL 등)을 가져와 2단계를 한국어 차트 도메인 데이터로 조금 더(QLoRA) 하는 것 = "도메인 적응". 채용 공고의 "멀티모달 VLM 최적화 및 도메인 적응(fine-tuning) 경험"이 정확히 이것.

왜 하필 차트가 어려운가

  • 픽셀 → 구조(축·눈금·범례·시리즈) → 수치 추론까지 다단계 해석이 필요
  • OCR + 공간 정렬("이 막대가 어느 라벨 것인가") + 산술이 동시에 요구됨
  • 한국어 라벨과 단위(억, 조)는 영어 중심 학습 데이터에서 희소 → 개선 여지가 큰 지점 (실제로 Day 1 진단에서 조→억 환산 10배 오류 발견)

용어 미니 사전

zero-shot
추가 학습 없이 사전학습된 모델 그대로 추론
SFT
Supervised Fine-Tuning. 정답이 있는 대화 데이터로 지도학습
LoRA / QLoRA
가중치 전체 대신 작은 저랭크 행렬만 학습하는 효율적 파인튜닝. QLoRA = 본체를 4bit 양자화해 메모리 절약 + LoRA
projector
비전 인코더 출력을 LLM 임베딩 공간으로 옮기는 변환기
dynamic resolution
이미지 크기에 따라 이미지 토큰 수가 달라지는 방식 (Qwen-VL 계열)
relaxed accuracy
수치 답변에 ±5% 오차를 허용하는 정답률. ChartQA 벤치마크의 표준 지표 — Day 3에서 구현한다
